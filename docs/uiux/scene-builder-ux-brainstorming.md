## Interview Findings

### 1. Real Usage Context

[事実] 依頼者は、通勤中や電車内などの隙間時間にiOSブラウザから開いて使う想定。

[事実] 電車内では声を出せない場面が多い。

[判断] Scene Builderは、音声前提のスピーキングアプリではなく、黙ってできるシチュエーション想像型の英作文/脳内会話トレーニングとして設計する。

### 2. First View and Theme Selection

[事実] 依頼者は、最初に軽くテーマを選びたい。

[判断] 初期表示で重いDeck管理を見せるのではなく、テーマチップから直感的に選べる入口にする。

[判断] テーマ選択後は、日本語の「言いたい文」ではなく、シチュエーションだけを出す。

### 3. Problem Style

[事実] 依頼者は、既存アプリのような日本語文の英訳ではなく、シチュエーションだけを見て自分で回答したい。

[判断] コア体験は「場面を想像する -> 自分で言いたいことを作る -> 選んだ難易度でAI添削する」。

[判断] `promptJa` のような日本語指示は補助情報として扱い、学習モードの主表示には出さない。

### 4. Difficulty Model

[事実] 依頼者は、難易度を自分で選びたい。

[判断] 難易度はAIが事後判定するものではなく、ユーザーが選ぶ練習モードとして扱う。

現時点の難易度モデル:

- `L1`: 動詞や動作を出せているか
- `L2`: 動詞に加えて、状態や感情を表す形容詞/語句を足せているか
- `L3`: シチュエーションに対して自然な短い返答になっているか
- `L4`: 会話として使える返答になっているか。理由、感想、相手への一言などが入っているか

[判断] L3/L4の単語数は依頼者が決め切らない。固定の単語数で合否判定せず、AIが選択難易度に対する成立度を評価する。

### 5. Hint Behavior

[事実] ヒントはデフォルト非表示にしたい。

[判断] ヒントは答えではなく、会話を作る材料として出す。

ヒントの例:

- 動詞
- 形容詞
- idiom / phrase
- 会話の方向
- L3/L4で足せる理由、感想、質問

### 6. AI Review and Score Meaning

[事実] 回答を書いたら `AI添削` をクリックする流れにしたい。

[事実] `10点満点` は英語力そのものの点数ではなく、選んだ難易度に対して今回の回答がどれくらい成立しているかの点数にする。

[判断] `AI添削` がMVPの主要保存イベントになる。

MVPで保存するもの:

- 最新回答
- 最新スコア
- 最新AI添削結果

Post-MVPで扱うもの:

- `AI添削` 実行ごとの履歴
- 過去回答
- 過去スコア
- スコア推移

### 7. AI Review Output

[事実] 必須項目は点数。

[判断] 結果画面は通勤中に読める軽さを優先する。

出力方針:

- 必須: 点数
- 基本表示: 自然な言い換え
- 任意表示: 修正するならこの文
- 任意表示: 次に足すとよい表現

[判断] 修正文、自然な言い換え、次に足す表現は重複しやすい。AIが提案でき、他項目と重複しない場合だけ出す。

### 8. Learning Mode and Creation Mode

[事実] Ownerのカード生成機能は残したい。

[事実] 作成モードでは、「このシチュエーションのときなんて言うんだろ」という気づきや疑問をラフに入力したい。

[事実] ラフな日本語の気づきを入力すると、AIがテーマ、シチュエーション、ヒント用の動詞/形容詞/表現、難易度別の評価観点を作る想定。

[事実] AI生成後は、保存前に確認/微修正し、`保存して学習に追加` する操作手順にする。

[判断] Owner toolsは単なる管理機能ではなく、`作成モード` としてプロダクトの一部に昇格する。

学習サイクル:

1. 気づく
2. 作成モードでラフに入力する
3. AIが練習用シチュエーションへ整える
4. 確認/微修正して保存する
5. 学習モードで出題される
6. AI添削で最新結果が保存される

## Decisions from Interview

| Topic | Decision |
|---|---|
| Primary context | 通勤などの隙間時間、iOSブラウザ、声を出さない利用 |
| First action | 軽いテーマ選択 |
| Problem display | シチュエーションだけを主表示する |
| Difficulty | ユーザーが自分で選ぶ |
| Hints | デフォルト非表示、クリックで表示 |
| Score | 選んだ難易度に対する成立度 |
| MVP persistence | 最新回答、最新スコア、最新AI添削結果 |
| Post-MVP | AI添削実行ごとの履歴と過去スコアUI |
| Modes | 学習モードと作成モードを用意する |
| Creation | ラフな日本語の気づきをAIが練習カード化し、確認後に保存する |

## Ideas Considered

| Idea | Summary | Decision |
|---|---|---|
| Today-first only | アプリ側が今日の1枚を出す | 却下。依頼者は最初に軽くテーマを選びたい |
| Translation prompt | 日本語文を出して英訳する | 却下。既存アプリとの差別化が弱い |
| Scenario-first challenge | シチュエーションだけを出して自分で会話を作る | 採用 |
| Fixed word count rubric | L3/L4を単語数で判定する | 却下。AIが難易度に対する成立度を評価する |
| AI review history in MVP | 毎回の添削履歴を初回実装で保存/表示する | Post-MVPに送る |
| Owner tools as admin box | 作成機能を管理UIとして残す | 却下。作成モードとして扱う |

## First PR Implications

最初のリデザインPRで扱う:

- 学習モード/作成モード切り替え
- テーマ選択を学習入口にする
- シチュエーション主表示
- ヒントをデフォルト非表示にする
- AI添削結果の表示を点数中心に軽くする
- 最新スコア/最新AI添削結果の保存を既存 `practice_records.review` に寄せる

最初のリデザインPRで扱わない:

- 添削履歴テーブル
- スコア推移UI
- ユーザー作成Deckの永続モデル
- 大規模なroute分割
