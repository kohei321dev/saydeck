# ADR 0001: Static-first learning app

- Status: Proposed
- Date: 2026-05-24

## Context

このプロジェクトの目的は、英検3級程度の学習者が日常会話で使える英文を増やすこと。特に、スケボー中に外国人の友達と話す場面で、自分の経験、質問、感想を短く言えるようにする。

リアルタイムAI添削は便利だが、最初からAPI連携を入れると費用、secret管理、失敗時の切り分けが増える。まず必要なのは、トピックカード、日記プロンプト、難易度別の模範回答を蓄積し、繰り返し見返せる状態を作ること。

## Decision

初期実装は、CSV/JSONなどの静的データを読み込むWebアプリとして作る。

- 教材データはrepo内で管理する
- 最初は静的な模範回答とセルフチェックを表示する
- ユーザー入力はブラウザ内に保存する
- AI採点は後続ADRで検討する
- Vercelなどへのdeployは、静的サイトとして始める

## Consequences

### Positive

- すぐ作れる
- GitHubで差分管理しやすい
- 教材の追加と修正をreviewしやすい
- API keyやsecretを初期段階で扱わなくてよい
- Vercelの無料枠に乗せやすい

### Negative

- 自由入力への自然な採点はできない
- 添削品質は模範回答とセルフチェックに依存する
- 学習履歴は最初は端末ごとのlocalStorageになり、複数端末同期はできない

## Follow-up

AI添削が必要になったら、以下を別ADRで決める。

- AI API provider
- コスト上限
- secret管理方法
- 入力データの保存有無
- 採点rubric
- fallback時の表示
